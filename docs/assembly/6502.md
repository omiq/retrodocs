# 6502 assembly

The IDE supports several **6502-family** assemblers, commonly including:

| Assembler | Typical use |
|-----------|-------------|
| **DASM** | Classic multi-platform 6502 assembler; common for Atari 2600 tutorials |
| **KickAss** | C64-focused; powerful macros and segments |
| **ACME** | Portable syntax; used in several cores |
| **ca65** (cc65) | Part of the **cc65** suite; pairs with **ld65** linking |

Which **exact** assembler appears depends on the **platform + preset** you select.

## Workflow

1. Pick a **6502** platform (C64, NES, Atari 8-bit, …).
2. Choose the **assembly** toolchain offered for that platform.
3. Edit `.asm` / `.s` sources per the template; **build** runs the assembler and often a **link** step.
4. Use **listing / symbol** output if your preset exposes it (KickAss-style symbol files, etc.).

## Concepts to teach early

| Topic | Why it matters |
|-------|----------------|
| **Registers** | A, X, Y — almost all 6502 logic |
| **Addressing modes** | `immediate`, `zero page`, `absolute`, `(indirect),Y`, … |
| **Memory map** | ROM vs RAM, I/O; **different per machine** |
| **Vectors** | Reset/NMI/IRQ — platform-specific |

## Pairing with high-level tools

- **cc65**: write C and **assembly** in one project; call between C and asm via **calling conventions** ([cc65 C notes](../c/cc65.md)).
- **BASIC `SYS`**: small ML routines from BASIC; must match load address ([Commodore BASIC](../basic/commodore-basic-v2.md)).
