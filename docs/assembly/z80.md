# Z80 assembly

**Z80** targets in the IDE typically use a **Z80 assembler** plus platform-specific **linking** or **binary layout** (cartridge, tape, disk image). The IDE wires these through **tool modules** (e.g. `z80` integration) for systems such as **MSX**, **Amstrad CPC**, and various **arcade** boards.

## Workflow

1. Select a **Z80**-based **platform** preset.
2. Open the **assembly** template; note **org / include** conventions for that core.
3. **Build**; fix errors from the assembler first, then runtime/emulator issues.

## Teaching order (suggested)

| Step | Content |
|------|---------|
| 1 | Registers **A, F, BC, DE, HL** and **alternates** on Z80 |
| 2 | **LD**, **ALU** ops, **JP/JR/CALL** — control flow |
| 3 | **Stack** — `PUSH/POP`, `SP` |
| 4 | **Platform I/O** — **completely different** per machine (MSX vs CPC vs arcade) |

!!! note "Platform first"
    Unlike 6502 home-computer tutorials that share “similar” I/O, Z80 systems **diverge quickly**. Tie each lesson to **one** platform preset until learners are comfortable.

## Toolchains

Exact assembler names depend on the preset (**z88dk** components, standalone asm, etc.). Document the **command-line** your IDE runs (build log) when you want this page to be precise.
