# C with cc65

**cc65** is a mature **C compiler** and development suite targeting **6502** systems (C64, Apple II, Atari 8-bit, NES with constraints, etc.). In the IDE it often appears as a **C** option for Commodore-class and related platforms.

## What’s in the box

| Piece | Role |
|-------|------|
| **cc65** | C compiler |
| **ca65** | Assembler (often used for startup + your `.s` files) |
| **ld65** | Linker — produces a binary for the target layout |
| **Runtime / libs** | Platform-specific; defines `main`, memory model, and I/O |

## Typical workflow in the IDE

1. Choose a **cc65**-capable **platform** preset.
2. Edit `main.c` (and sometimes `.s` / linker config) as the template shows.
3. **Build** — read errors from **cc65** first (types, prototypes), then **assembler/linker** messages.
4. Run in the **emulator**; use **debug** tools if the core supports them.

## C vs assembly

- Start with **small C** programs; drop to **asm** only for hot loops or hardware tricks.
- **Calling convention** matters when mixing **C and ca65** — consult **cc65** docs for your target.
- **Volatile** hardware registers need `volatile` or asm access patterns the optimizer won’t break.

## Memory and banking

Many targets have **bank switching** or **fixed ROM areas**. The **linker config** (`.cfg`) defines where code and data live — templates in the IDE encode the right defaults per platform.
